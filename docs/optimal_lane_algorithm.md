# 最適レーン配置分析のアルゴリズム解説

> 対象読者: 統計学の知識がない方。数式は最小限にし、日常の例え話で説明します。

---

## 1. この分析が解こうとしている問題

私たちのチームには7人のメンバーがいて、毎試合5人がそれぞれトップ・JG・ミッド・BOT・SUPの5つのロール（レーン）に入ります。

**「過去の試合データから、誰をどのレーンに置くのが一番勝てるか？」**

これがこの分析の目的です。一見すると「各メンバーの各ロールでの勝率を出して、一番高い組み合わせにすればいいのでは？」と思うかもしれません。しかし、そう単純にはいかない理由があります。

---

## 2. なぜ単純な勝率では不十分か — 「玉突き問題」

### 料理人の例え話

レストランに4人のシェフ A, B, C, D がいて、和食と洋食のどちらかを担当するとします。

- **シェフA が和食を担当したときの勝率（お客の評価）: 70%**
- **シェフB が和食を担当したときの勝率: 55%**

これだけ見ると「和食はシェフAに任せるべき」と思いますよね。

ところが、こんな裏事情があったらどうでしょう：

> シェフAが和食を作るとき、**たまたまいつも得意な洋食シェフCとペアになっていた**。
> シェフBが和食を作るとき、**Cは休みで、洋食が苦手なDがペアだった**。

つまり、シェフAの和食の高評価は「A自身の和食の腕」ではなく、「**ペアのCが洋食で稼いでくれた**」おかげかもしれないのです。

### LoL での具体例

Player_Cがミッドに行ったとき勝率49.6%、ジャングルに行ったとき勝率57.9%。単純に見ると「Player_CはJGがベスト」に見えます。

しかし、Player_CがJGに行く試合は、**たまたま他のメンバーも得意なロールに入っていた**試合が多かった可能性があります。逆にPlayer_Cがミッドのときは、**他の誰かが不慣れなロールに追い出されていた**かもしれません。

この「ある人がこのロールに入ると、別の人が違うロールに押し出される」連鎖を **玉突き効果** と呼んでいます。

**単純な勝率では「その人自身の能力」と「周りの配置の良し悪し」が混ざってしまう** のが問題です。

## 3. ロジスティック回帰 — 「各人の純粋な貢献」を推定する

### やりたいこと

「他のメンバーの配置を固定したとき、**この人がこのロールに入ることで勝率が何%変わるか**」を推定すること。

### どうやるか

まず、過去の全試合について「**誰がどのロールにいたか**」を 0/1 のフラグで記録します。

| 試合 | Player_C@TOP | Player_C@JG | Player_D@TOP | Player_E@JG | ... | 勝ち |
|------|:---------:|:--------:|:-------------:|:-----:|-----|:----:|
| 試合1 | 0 | 1 | 1 | 0 | ... | 1 |
| 試合2 | 1 | 0 | 0 | 1 | ... | 0 |
| ... | ... | ... | ... | ... | ... | ... |

次に、この表をもとに **「各フラグが勝率にどれだけ影響するか」を同時に推定** します。

これが「ロジスティック回帰」の役割です。

### ロジスティック回帰の直感的な理解

普通の回帰分析は「数値を予測する式」ですが、勝敗は0か1しかありません。「勝つ確率」は0%〜100%の範囲に収まる必要があります。

ロジスティック回帰は、内部で計算した値を **S字型のカーブ（ロジスティック関数）** に通すことで、出力を自動的に0〜100%の範囲に収めます。

```
勝つ確率(%) 
100 |                      ___________
    |                   __/
    |                 /
 50 |- - - - - - - -/- - - - - - - - -
    |              /
    |           __/
  0 |__________/
    ─────────────────────────────────
          内部スコア（低い ← → 高い）
```

内部スコアが高いほど勝つ確率が高く、低いほど負ける確率が高い。この内部スコアが「各メンバー×ロールのフラグの影響の足し算」で決まります。

重要なのは、**全メンバーのフラグを同時に入れて分析する** ことです。これにより「Player_CがJGに入っているときの勝率の高さ」のうち、「同時にPlayer_FがSUPにいたからの分」を分離して、**Player_C自身のJGとしての純粋な貢献** を推定できます。

## 4. Ridge 正則化 — 少ないデータで暴走しないためのブレーキ

### 問題

データが少ないとき（例：Player_GのJGは8試合しかない）、回帰分析は「たまたまの結果」を「真の能力」と勘違いしやすくなります。

コイン投げを8回して6回表が出たら、「このコインは75%で表が出る」と結論づけますか？ 8回では偶然のブレが大きすぎて、信用できないでしょう。

### Ridge 正則化の役割

「**係数（＝各メンバー×ロールの影響度）をゼロに近づける力**」を加えます。

- データがたくさんある場合 → データの力が強いので、ブレーキの影響は小さい → 妥当な推定値が出る
- データが少ない場合 → ブレーキが効いて、極端な推定を抑える → 「よくわからないので0（＝影響なし）に近い」と控えめに判断

つまり、**「確信が持てないことには大胆な結論を出さない」ための安全装置** です。

### CV（クロスバリデーション）による自動調整

ブレーキの強さ（正則化パラメータ C）をどこに設定するかは重要です。強すぎると何も検出できず、弱すぎるとデータの偶然に振り回されます。

CV（クロスバリデーション）は、データを5つのグループに分けて：

1. 4グループでモデルを学習
2. 残り1グループで予測精度をテスト
3. これを5回繰り返す

ブレーキの強さを色々変えながらこの手順を実行し、**テスト精度が最も高くなる強さを自動的に選ぶ** 仕組みです。

## 5. 限界効果（pp）— 「結局、何%変わるの？」

回帰モデルが出す生の係数は、ロジスティック関数の内部スコアの値であり、そのままでは「勝率が何%変わる」とは読めません。

そこで **限界効果** に変換します。これは「**他の配置を固定したとき、この人がこのロールに入ることで勝率が何パーセントポイント（pp）変わるか**」を示す値です。

例：
- Player_F@SUP の独立効果 = **+9.2pp**
  → 他の4人の配置が同じでも、SUPにPlayer_Fがいるだけで勝率が約9.2%ポイント上がる
- Player_C@TOP の独立効果 = **-15.5pp**
  → Player_CがTOPに入ると、それだけで勝率が約15.5%ポイント下がる

この値は「玉突き効果を分離済み」なので、単純勝率よりも「その人自身のロール適性」をより正確に反映しています。

## 6. Bootstrap 信頼区間 — 「答えのブレ幅」を測る

### なぜ必要か

473試合のデータから「Player_F@SUP = +9.2pp」と推定されましたが、もし別の473試合分のデータがあったら、結果は変わるかもしれません。+5ppかもしれないし、+13ppかもしれない。

**推定値がどれくらいブレうるか（不確実性）** を知らないと、その数字をどこまで信じていいかわかりません。

### やり方

1. 手元の473試合から、**ランダムに473試合を「重複あり」で抽出** する（同じ試合が2回選ばれることもある）
2. この新しいデータセットで回帰分析をやり直し、各メンバー×ロールの効果を記録
3. これを **500回繰り返す**
4. 500回の結果を並べて、下から2.5%目と97.5%目の値を取る → これが **95%信頼区間**

### 読み方の例

> Player_F@SUP: +9.2pp **[+4.8, +13.6]**

これは「500回やり直した結果、95%のケースでPlayer_FのSUP効果は+4.8ppから+13.6ppの間に収まった」ということです。

- **区間が狭い** → 推定が安定している → 信頼できる
- **区間が広い** → 推定が不安定 → 参考程度に留める
- **区間が0をまたぐ** (例: [-3, +8]) → プラスかマイナスかすら確定できない → あまり信じない方がいい

---

## 7. CV AUC — モデル全体の信頼度

### AUC とは

「**モデルが勝ち試合と負け試合をどれだけ正しく見分けられるか**」を測る指標です。

- **AUC = 0.50**: ランダム（コイン投げ）と同じ。モデルの予測は全く役に立たない
- **AUC = 0.60〜0.65**: ある程度の傾向は捉えている
- **AUC = 0.70以上**: かなり良い（LoLの勝敗は運要素も大きいので、ここまでは難しい）
- **AUC = 1.00**: 完璧に予測できる（現実にはありえない）

### どう使うか

CV AUC が 0.55 未満の場合、「このモデルはほぼ何も区別できていない」ということなので、推奨配置の信頼性も低いと判断してください。

0.60 前後であれば「傾向は捉えているが確実ではない」、0.65 以上であれば「一定の信頼を置いていい」と考えてください。

## 8. この分析の限界

### 限界1: データ数の制約

473試合で7人×5ロール＝最大35個のパラメータを推定しようとしています。統計学の経験則では、パラメータ1つあたり最低10〜20個のデータポイントが望ましいとされています。

現在のデータ量はこの基準でギリギリです。特に試合数が少ないメンバー×ロール（20試合未満）の推定は不安定です。

### 限界2: 加法性の仮定

このモデルは「各メンバー×ロールの効果を足し合わせる」ことで勝率を予測します。しかし実際には、特定の2人の組み合わせによるシナジー（相乗効果）が存在する可能性があります。

例えば「Player_EがJGでPlayer_Cがミッドのときだけガンクが噛み合って特に勝てる」というシナジーは、このモデルでは捉えられません。

### 限界3: 外挿のリスク

推奨された最適配置（例：「第1位」の配置）が、実際にプレイされたことがない場合があります。これは「モデルの予測」であって「実際の結果」ではありません。

未観測のパターンへの予測は「外挿（データの範囲外への推測）」であり、実測データよりも信頼性が低くなります。スクリプトでは外挿検知の警告を表示しています。

### 限界4: メタの変化

LoL のメタ（ゲームのバランスやプレイスタイルの流行）は時間とともに変化します。半年前に有効だった配置が今も有効とは限りません。全期間のデータを使うため、古いデータの影響が含まれます。

## 用語集

| 用語 | 意味 |
|------|------|
| **pp（パーセントポイント）** | 勝率の変化量。勝率が50%→55%なら「+5pp」 |
| **限界効果** | 他の条件を固定したとき、1つの要因を変えることで結果がどれだけ変わるか |
| **Ridge 正則化** | 推定が極端にならないようにするブレーキ |
| **CV（クロスバリデーション）** | データを分割してモデルの予測精度を公平に評価する方法 |
| **Bootstrap** | データから繰り返しサンプリングして推定のブレ幅を測る方法 |
| **AUC** | モデルの識別力の指標（0.5=ランダム、1.0=完璧） |
| **信頼区間 (95%CI)** | 「真の値がこの範囲に入っている」と95%の確信で言える範囲 |
| **外挿** | データの範囲外への推測。データ内の推測より信頼性が低い |
| **玉突き効果** | ある人がロールAに入ることで、別の人がロールBに押し出される連鎖 |
