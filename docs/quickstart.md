# ヨネでもわかるセットアップガイド

> **「……コマンド？ API？ それは……剣で斬れるのか？」**
>
> 大丈夫。このガイドは **書いてある通りにやるだけ** で動きます。
> 難しいことは何も考えなくていい。ヨネでもできた。お前にもできる。
>
> もっと詳しく知りたくなったら → [README](../README.md) へ

---

## まず全体像 — たった4ステップ

```
① Python を入れる        ← ソフトを1個入れるだけ
② API キーを取る         ← サイトにログインしてコピーするだけ
③ 設定ファイルを書く      ← メモ帳で3か所書くだけ
④ コマンドを2つ打つ      ← コピペするだけ
                         → 分析できる！
```

これだけ。本当にこれだけ。

---

## ① Python を入れる

Python はこのツールを動かすためのソフトです。LoL でいうクライアントみたいなもの。

1. ここを開く → https://www.python.org/downloads/
2. 黄色いでかいボタン **「Download Python 3.xx.x」** を押す
3. ダウンロードしたファイルをダブルクリックして開く
4. **最初の画面で「Add python.exe to PATH」にチェックを入れる**

```
  ┌─────────────────────────────────────────┐
  │  Install Python 3.xx.x                  │
  │                                         │
  │  ☑ Add python.exe to PATH  ← ★ここ！★  │
  │                                         │
  │  [ Install Now ]                        │
  └─────────────────────────────────────────┘
```

5. 「Install Now」を押す。終わるまで待つ

### ちゃんと入ったか確認する

画面左下の Windows マークを右クリック → **「ターミナル」** を開いて：

```
python --version
```

と打って Enter。`Python 3.xx.x` と出ればOK。出なかったら下の「よくあるトラブル」を見て。

## ② Riot API キーを取る

API キーは「Riot のサーバーから試合データをもらうための許可証」です。

1. ここを開く → https://developer.riotgames.com/
2. **LoL で使ってる Riot アカウントでログイン**する
3. ログインするとページに長い文字列が表示される

```
  RGAPI-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

4. これを **まるごとコピー** する（あとで貼る）

> **この許可証は 24時間で期限切れになります。**
> 次に使うときは同じページで **「REGENERATE API KEY」** を押して新しいのを取ってください。
> 毎回これだけやればOK。面倒だけどしょうがない。

## ③ 設定ファイルを書く

### やること 3-1：ファイルをコピーする

`config` フォルダの中にある `settings.yaml.example` というファイルを、**同じフォルダ内に** コピーして、名前を `settings.yaml` に変えてください。

ターミナルでやる場合はこれをコピペ：

```
cd "D:\データLoL"
copy config\settings.yaml.example config\settings.yaml
```

> エクスプローラーで右クリック → コピー → 貼り付け → 名前変更、でもOK

### やること 3-2：API キーを貼る

いまコピーした `config/settings.yaml` をメモ帳で開いてください。
（ファイルを右クリック →「プログラムから開く」→「メモ帳」）

中にこういう部分があります：

```yaml
riot_api:
  api_key: "RGAPI-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

`RGAPI-xxxxxxxx...` の部分を、**さっきコピーした自分の API キーに置き換えて**ください。
ダブルクォーテーション（`"`）は消さないで！キーだけ入れ替える！

```yaml
riot_api:
  api_key: "RGAPI-自分のキーをここに貼る"
  region: "jp1"       ← ここはそのまま触らない
  routing: "asia"     ← ここもそのまま
```

### やること 3-3：メンバーを登録する

同じファイルをもう少し下にスクロールすると `members:` という部分があります。
ここにフレックスで一緒に遊ぶメンバーの名前を書きます。

LoL クライアントの右上に表示されてる名前が `たろう#JP1` なら：

```yaml
members:
  - game_name: "たろう"     ← # の前の部分
    tag_line: "JP1"         ← # の後ろの部分
```

メンバーが複数いるなら、こんな感じで並べて書く：

```yaml
members:
  - game_name: "たろう"
    tag_line: "JP1"
  - game_name: "はなこ"
    tag_line: "JP1"
  - game_name: "ヨネ使い"
    tag_line: "JP1"
```

### やること 3-4：いつからの試合を取るか決める

同じファイルをもう少し下にスクロールすると `collection:` の中に `start_date` があります。

```yaml
collection:
  start_date: "2025-01-01"   ← この日付より後の試合を取ってくる
```

**「最近のだけでいいや」** という人は `"2025-06-01"` とかに変えればOK。
古い日付にすると取ってくる量が多くなって時間がかかるけど、その分たくさん分析できる。

> よくわからなければそのままでOK。あとから変えてもう1回取り直せる。

書いたら **上書き保存** して閉じる。

## ④ コマンドを打つ（コピペでOK）

ターミナル（さっき Python の確認で開いたやつ）に、以下を **上から順番に** コピペして Enter。

### まず最初の1回だけ — 必要な道具を入れる

```
cd "D:\データLoL"
pip install -r requirements.txt
```

ズラズラ文字が出るけど気にしないで待つ。

### 試合データを取ってくる

```
python src/collect.py
```

数分かかる。じっと待つ。LoL の試合データを全部ダウンロードしてる最中。

### データを変換する

```
python src/process.py
```

一瞬で終わる。

### 成績を見る！

```
python src/analyze.py overview
```

**おめでとう！** チーム全体の勝率やメンバー別の成績が表示されたはず。
ヨネにもできた。……救えたはずだ。

## 2回目以降 — 新しい試合を取り込むとき

試合のあとにデータを更新したくなったら、ターミナルでこの2つを打つだけ：

```
cd "D:\データLoL"
python src/collect.py
python src/process.py
```

これで新しい試合が追加される。あとは好きな分析コマンドを打てばいい。

## よくあるトラブル

### 「API キーが無効です」と出た

キーは24時間で切れる。 https://developer.riotgames.com/ で
**「REGENERATE API KEY」** を押して新しいキーを取って、`config/settings.yaml` に貼り直す。

### 「python が見つかりません」と出た

Python を入れたとき **「Add python.exe to PATH」にチェックし忘れた**。
Python をアンインストールしてから、もう一度インストールし直す（今度はチェック忘れずに）。

### 「pip が見つかりません」と出た

代わりにこっちを試す：

```
python -m pip install -r requirements.txt
```

### ここに書いてないエラーが出た

→ **[README.md](../README.md)** に詳しいことが全部書いてある！

### 「同ランク帯と比べてどう？」系のスクリプトが動かない

`benchmark_comparison.py` や `champion_benchmark.py` など、**平均的なプレイヤーとの比較**をするスクリプトは、別途ベンチマークデータの収集が必要。上の基本セットアップだけでは入っていない。

やり方は **[README のベンチマークデータの収集](../README.md#ベンチマークデータの収集任意)** を見てね。
やること自体は簡単で、コマンドを2つ打つだけ。
